# 企业级新项目门禁

企业级新产品要加这些门禁，但不要把日常开发变成仪式。

## 1. 需求门禁

- 每个里程碑都有目标、非目标、用户、验收标准和上线负责人。
- 第一个版本先跑通纵向切片，再扩展框架和流程。

## 2. 非功能基线

- 安全：
- 审计：
- 可观测性：日志 / 指标 / 链路
- 备份恢复：
- 性能预算：
- 可用性：
- 可访问性：
- 成本预期：

## 3. 权限门禁

- 角色：
- 资源：
- 动作：
- 数据范围：
- 审计链路：

## 4. 数据门禁

- 数据模型：
- 索引：
- 迁移：
- 回滚：
- 种子数据：
- 保留策略：
- 隐私要求：

## 5. API 门禁

- 契约：
- 错误格式：
- 兼容性：
- 幂等：
- 分页：
- 版本策略：

## 6. 环境和交付门禁

- dev/staging/production 一致性：
- 配置 owner：
- 密钥管理：
- CI 必跑检查：
- 发布产物可追溯：
- 回滚触发条件：
- 发布后观察窗口：
