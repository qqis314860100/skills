# atom-team-workflow

面向个人或小团队的软件交付工作流 skill，适合前端 1 人、后端 1 人、AI 辅助开发、老项目重构和企业级新项目从 0 到 1。

## 适合用在

- 个人项目或小团队开发流程设计
- 前端 1 人 + 后端 1 人协作
- 企业级老项目重构
- 新项目启动和项目基线设计
- 技术栈升级、性能优化、UI 改版、功能改造
- 提交/PR 规则、验证规则、发布回滚规则
- AI agent 参与开发时的边界和验收标准

## 触发关键词

显式触发最稳：

```text
使用 $atom-team-workflow 帮我设计这个项目的重构工作流
```

常见触发词：

- 原子团队工作流
- 个人开发工作流
- 小团队开发工作流
- 前端 1 人后端 1 人
- 1+1 开发模式
- AI 辅助开发流程
- 老项目重构
- 企业级老项目重构
- 新项目开发流程
- 从零开发企业级项目
- 技术现代化
- 技术栈升级
- 性能优化流程
- UI 改版流程
- 功能切片
- 验证规则
- 提交规则
- PR 规则

- 轻量护栏
- 减少流程负担

## 核心原则

```text
日常轻、边界硬、高风险重、上下文省、自动化单独管。
```

## 注意

这个 skill 是通用工作流底座，不替代每家公司自己的业务规则、上线制度、安全合规要求和技术栈约束。
